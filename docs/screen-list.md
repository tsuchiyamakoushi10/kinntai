# 画面一覧

画面 ID は `S-<カテゴリ>-<連番>` 形式。カテゴリは `C`（共通）、`E`（従業員）、`A`（管理者）、`T`（共有タブレット専用）。

---

## 1. 共通画面

| ID | 画面名 | 主な用途 | フェーズ |
| --- | --- | --- | --- |
| S-C-01 | ログイン | メール + パスワードでログイン | MVP |
| S-C-02 | パスワード再設定リクエスト | メールアドレス入力→リセットリンク送付 | MVP |
| S-C-03 | パスワード再設定 | 新パスワード入力 | MVP |
| S-C-04 | エラー / 権限なし | 共通エラーページ | MVP |

---

## 2. 共有タブレット専用画面

事業所に設置するタブレット用。常時ログイン状態を保持し、画面遷移は最小限。

| ID | 画面名 | 主な操作 | フェーズ |
| --- | --- | --- | --- |
| S-T-01 | タブレット初期セットアップ | 拠点を選び、端末を登録（初回のみ） | MVP |
| S-T-02 | 本人選択 | 氏名一覧から自分を選択。50音 / 拠点でフィルタ | MVP |
| S-T-03 | 暗証番号入力 | 4桁の暗証番号で本人確認 | MVP |
| S-T-04 | 打刻メニュー | 大きな4ボタン（出勤 / 退勤 / 休憩開始 / 休憩終了）、直近打刻表示 | MVP |
| S-T-05 | 打刻完了 | 「打刻しました」3秒表示で S-T-02 に自動戻り | MVP |

### 操作フロー（共有タブレット）
```
S-T-02 (本人選択)
  ↓ 氏名タップ
S-T-03 (暗証番号)
  ↓ 4桁入力
S-T-04 (打刻メニュー)
  ↓ ボタンタップ
S-T-05 (完了)
  ↓ 自動遷移
S-T-02 へ戻る
```

## 5. 画面設計の方針

- **従業員側のホーム（S-E-01）と共有タブレット打刻（S-T-04）は、ボタンを大きく、原則1タップ完結**。
- **管理者の勤務表編集（S-A-08）はキーボード操作も意識**する。1か月分を一気に編集するため、矢印キー移動・コピー&ペースト対応を MVP で入れる。
- **エラーメッセージは現場の言葉で**。「Validation error: required field」のような開発者向け表現を出さない。
- **権限不足は静かに非表示**。従業員にメニュー項目自体を出さないことを優先し、ガード用画面（S-C-04）はバックアップ。
