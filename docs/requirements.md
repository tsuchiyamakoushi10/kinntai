# 要件定義書

## 1. プロジェクトの目的

小規模〜中規模の介護事業所が、**勤怠管理・勤務表作成・有給管理・従業員情報管理** を一気通貫で行える Web アプリを提供する。

主なユーザーは管理者（事業所長・サービス提供責任者など）と現場の介護職員。専用ハードウェアを購入することなく、既存の **共有タブレット** と **個人スマホ** だけで運用できることを重視する。

## 2. 想定規模（初期）

| 項目                      | 想定値                                                                                                      |
| ------------------------- | ----------------------------------------------------------------------------------------------------------- |
| 拠点数                    | 5拠点（ナーシングホーム結いの心 / デイサービス結いの心 / ショートステイ結いの心 / デイサービス梨花 / 厨房） |
| 従業員数                  | 約55名（全拠点合計、正社員30 + パート25）                                                                   |
| 1拠点あたり同時打刻ピーク | 10〜15名程度（朝の出勤集中時、24h 施設の夜勤交代含む）                                                      |
| 管理者ユーザー数          | 数名〜10名以内                                                                                              |
| シフトパターン数          | 約30種（拠点専用パターン多数。詳細は [`docs/shift-patterns.md`](shift-patterns.md)）                        |

## 3. ユーザーロール

| ロール                               | 想定する人                                                      | 主な権限                                                     |
| ------------------------------------ | --------------------------------------------------------------- | ------------------------------------------------------------ |
| **管理者**                           | 事業所長、事業所責任者、サービス提供責任者、管理事務            | 全拠点の従業員管理、勤務表編集、勤怠承認、有給付与、各種設定 |
| **拠点責任者**（Phase 2 で分離検討） | 拠点リーダー                                                    | 自拠点のみの勤怠承認・勤務表編集                             |
| **従業員**                           | 正社員、契約社員、パート、夜勤スタッフ など、現場で勤務する全員 | 自分の勤怠閲覧、打刻、希望休 / 修正申請（Phase 2）           |

> **MVP では「管理者」と「従業員」の 2 ロールのみ**。「事業所責任者」は **管理者の一種** として扱う。拠点責任者ロールの分離は Phase 2 で再検討する。ロール別の権限詳細は [`user-roles.md`](user-roles.md) を参照。

## 4. 機能要件

### 4.1 MVP（フェーズ1）

#### 認証

- 管理者ログイン（メール + パスワード）
- 従業員ログイン（メール + パスワード、または 従業員コード + 暗証番号）
- 共有タブレット用「打刻専用モード」: ログイン状態を維持しつつ、本人を暗証番号で選択して打刻

#### 打刻

- 出勤打刻
- 退勤打刻
- 休憩開始
- 休憩終了
- 打刻時に拠点を自動判定（タブレットの場合は拠点固定、スマホの場合は所属拠点をデフォルト）
- 打刻履歴の即時表示（直近5件程度）

#### 勤怠閲覧

- 従業員: 自分の月別勤怠（出勤・退勤・休憩・実労働時間・残業）を一覧表示
- 管理者: 拠点 / 個人別の月別勤怠を一覧表示、未承認の打刻をハイライト

#### 勤怠確認・承認

- 管理者が月締めで一括または個別に勤怠を承認
- 承認後は従業員側からの修正不可（修正は管理者経由）

#### 月別勤怠集計

- 出勤日数、実労働時間、所定内 / 所定外、深夜時間、休日労働、有休消化日数
- 画面上で表示（CSV出力は Phase 2）

#### 勤務表作成

- 月次の勤務表ビュー（縦: 従業員、横: 日付）
- セルにシフトパターン（早番・日勤・遅番・夜勤・明け・公休・有休・希望休）を割当
- 拠点ごとに作成
- コピー機能（前月をコピーして編集）

#### シフトパターン設定

- 名称、開始時刻、終了時刻、休憩時間、種別（早 / 日勤 / 遅 / 夜入 / 夜明 / 半日勤務 / 拠点固有パターン / 休み系）
- カラー設定（勤務表上の視認性のため）
- 拠点固有 / 全拠点共通の切り替え（`shift_patterns.office_id` で表現）
- 複合パターン（午前有休 + 午後勤務など）に対応する `paid_leave_units` 値の設定（詳細は [`docs/shift-patterns.md`](shift-patterns.md)）

#### 従業員情報管理

- 基本情報: 氏名（漢字・かな）、生年月日、性別、電話番号、住所
- 緊急連絡先: 氏名、続柄、電話番号
- 雇用情報: 雇用形態、所属拠点、職種（介護職員 / 看護職員 / 生活相談員 / ケアマネ / 事務 など）、入社日、雇い入れ日、所定労働時間 / 日、所定労働日数 / 週、基礎時給または月給
- 在籍状態: 在籍 / 退職、退職日
- 資格情報: 介護福祉士、初任者研修、実務者研修などの保有資格と取得日・登録番号（将来のキャリアアップ助成金対応で利用）

#### 有給管理（雇い入れ日基準）

- 雇い入れ日から6か月経過時点で自動付与
- 以降、年次で自動付与（労基法準拠の日数テーブル）
- 所定労働日数に応じた比例付与（パート対応）
- 手動付与・調整・繰越・消滅（2年）の管理
- 残数を従業員ホームに常時表示
- **年5日取得義務チェック**: 年10日以上付与された従業員について、付与日から1年以内の取得日数が5日に満たない場合に管理者画面でアラート表示

### 4.2 Phase 2（試験運用後）

- 打刻修正申請（従業員→管理者）と承認フロー
- 希望休申請と勤務表への反映
- 月次集計の CSV / Excel 出力
- 給与計算ソフト向け CSV エクスポート（フォーマット未定）
- 拠点管理者ロールの分離と権限制御
- スマホでの打刻時の位置情報チェック（拠点近傍判定）
- 通知（打刻忘れアラート、申請通知）— メールまたは LINE

### 4.3 Future（中長期）

- IC カード打刻 / 生体認証打刻
- 雇用契約書の自動生成・電子署名
- 労働条件通知書の自動生成
- 社労士確認用 PDF 出力（月次勤怠・有給状況・労務情報をまとめた帳票）
- キャリアアップ助成金の申請に必要な情報集計・出力
- 処遇改善加算 / 特定処遇改善加算の算定支援
- マルチテナント化（複数法人での利用）
- 介護記録 / 訪問記録との連携

## 5. 非機能要件

| カテゴリ         | 要件                                                                    |
| ---------------- | ----------------------------------------------------------------------- |
| **同時利用**     | 同時打刻 30 req/s 程度を許容（朝のピーク × 安全係数）                   |
| **レスポンス**   | 打刻アクションは応答 1 秒以内を目標                                     |
| **稼働率**       | 99% 程度（小規模事業所向けのため、夜間メンテ可）                        |
| **データ保持**   | 勤怠データは最低 5 年保持（労基法 3 年 + 余裕）                         |
| **バックアップ** | 日次フルバックアップ、最低 30 日分保持                                  |
| **タイムゾーン** | サーバ・DB ともに UTC で保存、表示は `Asia/Tokyo`                       |
| **ブラウザ対応** | 最新版の Chrome / Safari / Edge。iPad Safari 必須                       |
| **画面サイズ**   | 共有タブレット（1024×768 横向き）と スマホ（375×667 縦向き）両対応      |
| **セキュリティ** | パスワード bcrypt ハッシュ、HTTPS 必須、CSRF 対策、ログイン試行回数制限 |
| **個人情報保護** | PII はログに出さない、エクスポート時に管理者ログを残す                  |

## 6. 法令・制度の前提

- **労働基準法**: 労働時間（1日8時間 / 週40時間）、割増賃金（深夜 25%、休日 35%、時間外 25%）、年次有給休暇の付与基準
- **介護保険法 / 介護報酬**: 直接の算定はしないが、将来の処遇改善加算対応のための情報項目を確保する
- **個人情報保護法**: 従業員情報の安全管理措置

法令準拠が関わるロジック（特に有給付与日数、深夜割増、夜勤跨ぎ判定）は、根拠とともにコード内コメントまたはドキュメント側に記録する。

## 7. スコープ外（明示）

以下は本アプリでは扱わない。

- 給与計算そのもの（時間集計までを行い、給与ソフトに連携する想定）
- シフト自動生成（AI 配置最適化など）
- 利用者（介護を受ける方）の情報管理
- 介護記録・サービス提供記録
- 請求業務（国保連連携など）
