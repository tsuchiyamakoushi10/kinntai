# デモ手順 (顧客確認用)

Phase 1 (MVP) の実装が概ね揃ったので、現状の画面を顧客に動かしてもらいながらフィードバックを集めるためのガイド。
自分の PC で `pnpm dev` を立ち上げ、隣で操作してもらう想定。

---

## 1. デモの目的

- **「現場の言葉になっているか」を確認してもらう**（ボタン文言、項目名、エラーメッセージ）
- **MVP のスコープに過不足がないか**を見てもらう
- 仕様書だけでは伝わりにくい **画面遷移の浅さ・1 タップ完結** を体感してもらう

逆に、今回は **見た目の細かい調整・色味・ロゴ** は議題にしない (Phase 2 以降で詰める)。

---

## 2. 起動 (デモ前に 1 回)

```bash
# 1. PostgreSQL を起動
pnpm db:up

# 2. マスタ + デモデータを投入（既にデータが入っていればスキップで OK）
pnpm db:seed          # 5 拠点 / 55 名 / 34 シフトパターン / admin & employee ユーザ
pnpm db:seed:demo     # 当月 1 か月分のシフト・打刻・有給付与を生成（冪等）

# 3. 開発サーバー
pnpm dev              # http://localhost:3000
```

> デモ中にやり直したくなったら `pnpm db:seed:demo` を再実行。打刻状態は当月の半分くらいが
> 自動的に「勤務中 / 退勤済」に分かれるので、ダッシュボードもそれっぽく見える。

## 4. デモシナリオ (推奨順)

### A. 管理者ダッシュボード (2〜3 分)

1. `/login` で管理者ログイン → `/admin` (ダッシュボード) に着く
2. **見せどころ**:
   - 拠点別の「在籍 / 勤務中 / 休憩中 / 退勤済」が一目で出る
   - 「年5日アラート」「承認待ち」がカード形式で目に入る
3. **聞きたい**: ここに常時出ていてほしい数字は他にあるか？

### B. シフト管理と勤務表 (3〜4 分)

1. サイドバー → **シフトパターン** (`/admin/shift-patterns`)
   - 早 / 日勤 / 遅 / 夜入 / 夜明 / 半日 …等 34 種が一覧で出る
   - パターンを 1 件ひらいて、「色 / 休憩時間 / 有給単位」を編集できることを見せる
2. サイドバー → **勤務表** (`/admin/shifts`)
   - 拠点を選ぶ → 縦に従業員、横に日付の月次グリッド
   - セルをクリックしてシフトを切り替えられること、矢印キーで移動できることを見せる
3. **聞きたい**: 現場で使っている紙 / Excel の勤務表と比べて足りないものは？

### C. 勤怠承認と月別集計 (3〜4 分)

1. サイドバー → **勤怠** (`/admin/attendance`)
   - 月単位で従業員一覧、実労働 / 残業 / 深夜 / 有休消化が出る
   - 1 行クリック → 個別の月次勤怠（日別の打刻一覧）
   - 「承認」ボタンで月締めできる
2. **聞きたい**: 残業の出し方（所定オーバー）はこの定義で合っているか？

### D. 有給管理 (2〜3 分)

1. サイドバー → **有給管理** (`/admin/leave`)
   - 法定通り、雇い入れ日基準で自動付与されている (15 年勤続 = 20 日 等)
   - 残数・有効期限が並ぶ
2. **年5日アラート** (`/admin/leave/alerts`)
   - 取得不足の従業員を抽出する画面（Phase 1 内で実装済）
3. **聞きたい**: 比例付与（パート）の表記、現場で違和感はないか？

### E. 共有タブレット打刻 (2 分)

1. 管理者でログインしたまま、別タブで `/tablet/setup`
   - 拠点を選んで「この拠点で使う」 → 以降この端末はその拠点扱い
2. `/tablet` → 50 音タブから本人を選ぶ
3. PIN 4 桁 → 大きな 4 ボタン（出勤 / 退勤 / 休憩開始 / 休憩終了）
4. 押すと「打刻しました」で 3 秒後に自動で本人選択に戻る
5. **聞きたい**: 文字 / ボタンの大きさは適切か。混雑する朝でも詰まらないか。

### F. 個人スマホからの打刻 (2 分)

1. 別ブラウザ / シークレットウィンドウで従業員ログイン (`e0001@kinntai.local`)
2. `/me` ホーム → 今日のステータス + 大きな打刻ボタン
3. 月別勤怠 (`/me/attendance`)、シフト (`/me/shifts`)、有給 (`/me/leave`) を巡る
4. **聞きたい**: 従業員が日常で見るのはこの 4 画面で足りるか。

## 5. 顧客に確認してほしい質問リスト

優先度の高い順に。

1. **業務用語**: 「拠点」「在籍」「勤務中」「年5日」など、現場で通じる言葉になっているか
2. **シフトパターン**: 34 種で抜けはないか（特に「梨5」「厨房 C」など拠点固有のもの）
3. **打刻の流れ**: 1 タップで完結するフローで、現場が受け入れられそうか
4. **承認フロー**: 月締めの「承認」ボタンが、社労士提出までの流れに合うか
5. **有給比例付与**: パートの比例付与表（厚労省準拠）の表示が、現場の説明に使えるか
6. **Phase 2 に積む候補**: 修正申請、希望休、CSV 出力、雇用契約書（このうち最優先は？）
