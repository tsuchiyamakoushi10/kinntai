## 2. シフトパターン分類

### 2.1 フル勤務系（24h 施設）

主にナーシング / ショートで使うフルタイム帯。

| code        | 名称 | 開始  | 終了  | 跨日 | 休憩(分) | 主な拠点               |
| ----------- | ---- | ----- | ----- | ---- | -------- | ---------------------- |
| `EARLY`     | 早   | 06:30 | 15:30 | ×    | 60       | NRS-CENTER, SHO-CENTER |
| `DAY`       | 日勤 | 08:15 | 17:15 | ×    | 60       | NRS-CENTER, SHO-CENTER |
| `LATE`      | 遅   | 10:00 | 19:00 | ×    | 60       | NRS-CENTER, SHO-CENTER |
| `NIGHT_IN`  | 夜入 | 16:30 | 24:00 | ×    | 30       | NRS-CENTER, SHO-CENTER |
| `NIGHT_OUT` | 夜明 | 00:00 | 08:30 | ×    | 30       | NRS-CENTER, SHO-CENTER |

> 夜勤は「夜入（当日）」と「夜明（翌日）」の 2 枚のシフトに分割して勤務表に貼る運用。`shift_kind` も `night_in` / `night_out` を別物として持つ。

### 2.2 フル勤務系（デイ / ショート専用）

時間帯は同じ 8:15–17:15 だが、サービス区分ごとに別 code で管理。

| code           | 名称     | 開始  | 終了  | 跨日 | 休憩(分) | 主な拠点              |
| -------------- | -------- | ----- | ----- | ---- | -------- | --------------------- |
| `DAY_CARE`     | デ日     | 08:15 | 17:15 | ×    | 60       | DAY-CENTER, DAY-RIKKA |
| `DAY_SHORT`    | ショ日   | 08:15 | 17:15 | ×    | 60       | SHO-CENTER            |
| `OFFICE_WORK`  | 事務     | 08:15 | 17:15 | ×    | 60       | 全拠点（管理者のみ）  |
| `RECORD_ROUND` | 実績周り | 08:15 | 17:15 | ×    | 60       | 全拠点（管理者のみ）  |

> **事務 / 実績周り**は管理者（施設管理者 = `employees.is_manager=true`）の勤務。日勤と同時間帯・同じく `shift_kind = work` で、午前 1・午後 1 としてフロア人数にもカウントする。管理者が S-E-10 のシフト希望画面で「事務日 / 実績周り日」を月あたり事務日 2 日・実績周り日 1 日まで指定し（`shift_preferences.office_day` / `record_round`）、自動作成がその日を固定配置して公休を入れない。

### 2.9 複合パターン（有休半休 + 勤務）

1 日の中で午前 / 午後に有休消化と勤務が同居するパターン。`shift_kind` は `work` 側で扱い、有休 0.5 日消化は `paid_leave_units` で機械判定する。

| code              | 名称  | 内容                           | 勤務時刻    | `paid_leave_units` |
| ----------------- | ----- | ------------------------------ | ----------- | ------------------ |
| `AM_LEAVE_PM_DAY` | 有/日 | 午前有休 + 午後 デ日相当の勤務 | 13:00–17:15 | 0.5                |
| `AM_DAY_PM_LEAVE` | 日/有 | 午前 デ日相当の勤務 + 午後有休 | 08:15–12:30 | 0.5                |

## 3. 新設・変更したフィールド

### 3.1 `shift_kind` enum の変更

`night` / `after_night` を **`night_in`** / **`night_out`** にリネーム。これにより「夜入」「夜明」を別パターンとして明示的に区別できる。

| 旧            | 新          |
| ------------- | ----------- |
| `night`       | `night_in`  |
| `after_night` | `night_out` |

### 3.2 `shift_patterns.paid_leave_units` を追加

複合パターン（有休 + 勤務）と純粋有休を機械判定するため、`shift_patterns` に **`paid_leave_units` (Decimal(3,1))** を追加。

| パターン                                                        | 値    |
| --------------------------------------------------------------- | ----- |
| 通常勤務 / 半日勤務 / 休み系（公休・欠勤・希望休）/ 夜入 / 夜明 | `0.0` |
| 純粋有休 (`PAID_LEAVE`)                                         | `1.0` |
| 複合 (`AM_LEAVE_PM_DAY` / `AM_DAY_PM_LEAVE`)                    | `0.5` |

シフトを 1 行割り当てるだけで「有休が何単位消化されたか」が確定するため、`paid_leave_consumptions` へのレコード追加もこの値を元に行う。

## 4. 要確認事項（運用に出る前にヒアリング必須）

1. **半日 A〜F、半日午後** はどの拠点で使うか
2. **半日E** の code（元データに表記なし、9:00-11:00 のパターン）の正式名
3. 早 / 日勤 / 遅 を **ナーシングとショート両方で同じ code で共有してよいか**（拠点ごとに別運用なら code を分ける）
4. 「夜入 16:30-24:00」「夜明 0:00-8:30」の時刻は固定か、施設ごとに前後するか
5. 厨房は土日も稼働するか（公休のカウントに影響）

これらは現場確認後に本ドキュメントを更新し、`master.ts` を再投入（`pnpm db:seed:master`）して反映する想定。
