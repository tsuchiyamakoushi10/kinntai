# ユーザーロール定義

本ドキュメントでは、kinntai に登場するユーザーロールと、各ロールが何をできて何をできないかを定義する。MVP では **管理者** と **従業員** の2ロールのみを実装する。

---

## 1. ロール一覧

| ロール ID     | 表示名     | 想定する人                                           | MVP 実装              | 備考                     |
| ------------- | ---------- | ---------------------------------------------------- | --------------------- | ------------------------ |
| `admin`       | 管理者     | 事業所長、事業所責任者、サービス提供責任者、管理事務 | ✅                    | 全拠点に対して権限を持つ |
| `branch_lead` | 拠点責任者 | 拠点リーダー、現場主任                               | ⏳ Phase 2 で分離検討 | MVP では `admin` に統合  |
| `employee`    | 従業員     | 正社員、契約社員、パート、夜勤スタッフ               | ✅                    | 自分自身に関する操作のみ |

> **ロールと「雇用形態」「職種」は別軸**。雇用形態（正社員 / 契約 / パート）は労務上の区分、職種（介護 / 看護 / 生活相談員 など）は業務区分、ロールはシステム上の権限区分。

---

## 2. 想定ユーザー像

### 2.1 管理者（`admin`）

- **事業所長 / 事業所責任者**: 全社的な意思決定、月次の勤怠承認、有給付与、人事情報の最終管理者
- **サービス提供責任者**: 訪問介護などで利用者対応の責任を持つ立場。シフト作成・調整を主導する
- **管理事務**: 給与計算前の勤怠締め、雇用契約関連、社労士・行政対応窓口

> 介護事業所の実情として、これらの役割は同一人物が兼ねるケースが多い。MVP では1つのロールでカバーする。

### 2.2 従業員（`employee`）

- **正社員**: フルタイム、月給。早番〜夜勤を含むシフトに入る
- **契約社員**: 期間定めあり、フルタイムまたは短時間
- **パート**: 時給制、短時間。週所定労働日数に応じた比例付与の対象
- **夜勤スタッフ**: 夜勤専門の従業員。雇用形態は問わない（正社員でもパートでもありうる）

> 「夜勤スタッフ」はロールではなく **シフトの組み方** の問題。システム上は他の従業員と同じ `employee` ロールで扱う。

### 2.3 拠点責任者（`branch_lead`、Phase 2 想定）

- 自拠点に限定して、勤怠承認・勤務表編集・希望休承認を行う
- 他拠点の情報は閲覧不可
- 従業員の新規登録・有給付与など人事的操作は不可（管理者経由）

## 3. 権限マトリクス

`R`: 閲覧可、`W`: 編集可、`A`: 承認権限、`-`: 不可、`SELF`: 自分の分のみ

### 3.1 自分の勤怠・シフト・有給

| 操作                          | 管理者 | 拠点責任者 | 従業員 |
| ----------------------------- | ------ | ---------- | ------ |
| 自分の打刻                    | W      | W          | W      |
| 自分の勤怠閲覧                | R      | R          | R      |
| 自分のシフト閲覧              | R      | R          | R      |
| 自分の有給残数閲覧            | R      | R          | R      |
| 自分の打刻修正申請（Phase 2） | W      | W          | W      |
| 自分の希望休申請（Phase 2）   | W      | W          | W      |

### 3.2 他者の勤怠・シフト・有給

| 操作                          | 管理者 | 拠点責任者      | 従業員 |
| ----------------------------- | ------ | --------------- | ------ |
| 他者の勤怠閲覧                | R      | R（自拠点のみ） | -      |
| 他者の勤怠承認                | A      | A（自拠点のみ） | -      |
| 他者のシフト編集              | W      | W（自拠点のみ） | -      |
| 他者の有給付与・調整          | W      | -               | -      |
| 年5日取得アラート閲覧         | R      | R（自拠点のみ） | -      |
| 打刻修正申請の承認（Phase 2） | A      | A（自拠点のみ） | -      |
| 希望休申請の承認（Phase 2）   | A      | A（自拠点のみ） | -      |

### 3.3 マスタ管理

| 操作                    | 管理者 | 拠点責任者 | 従業員 |
| ----------------------- | ------ | ---------- | ------ |
| 従業員 新規登録・編集   | W      | -          | -      |
| 従業員 退職処理         | W      | -          | -      |
| 拠点 追加・編集         | W      | -          | -      |
| シフトパターン 編集     | W      | R          | R      |
| ユーザーアカウント 発行 | W      | -          | -      |

### 3.4 出力・エクスポート（Phase 2 以降）

| 操作                                       | 管理者 | 拠点責任者      | 従業員 |
| ------------------------------------------ | ------ | --------------- | ------ |
| 月次集計 CSV 出力                          | W      | W（自拠点のみ） | -      |
| 給与ソフト連携 CSV 出力                    | W      | -               | -      |
| 雇用契約書 / 労働条件通知書 出力（Future） | W      | -               | -      |
| 社労士確認用 PDF 出力（Future）            | W      | -               | -      |

## 5. 実装メモ

- ロールは `users.role` カラム（`admin` / `employee`）で表現。`branch_lead` は Phase 2 で追加。
- 「自拠点のみ」の制御は、Phase 2 で `user_office_assignments` 中間テーブルを追加して実現する想定。MVP では `admin` は全拠点アクセス、`employee` は自分自身のデータのみという単純な制御で十分。
- 認可チェックは Server Action / Route Handler の入口で必ず行う。UI 側の表示制御だけに頼らない。
- 管理者が誤って自身を退職処理 / アカウント無効化できないよう、自分自身に対する破壊的操作は禁止する。
